**学习循环（核心）**：

```
进入单元 (P02)
  │
  ▼
┌───── 循环 N 个字 ─────────────────┐
│  P03 认  →  P04 写  →  P05 练     │
│  (单字 5~10s 速练，错也过)         │
└──┬───────────────────────────────┘
   │ 一组字（≈5字）学完
   ▼
P06 小组练习 (2~3题，听音/看图/拖拽)
   │
   ├── 还有下一组 ──→ 回到 P03 学新字
   │
   └── 全单元字学完
          ▼
       P07 单元测验 (8~10题综合)
          │
          ▼
       P08 测验结果
          │
       ┌──┴──┐
       ▼     ▼
     通过   未通过
       │     │
       │     └──→ "再试一次" → 回 P07
       │
       ├──→ 解锁下一单元 → 回 P01
       └──→ 进入 Unit-02 时若未付费 → P09 付费引导
```

**过关庆祝（P07-celeb）**：每个字"练"完成 → 触发庆祝小弹层/页（**注意：当前设计稿"P07-过关庆祝页"是单字过关，与PRD"P07单元测验"编号冲突，详见偏离清单**）。

## 3. 偏离清单 ✅ 已全部清零（2026-04-20）

> 历史决策记录，作为后续决策的参考案例。

### ✅ #1：P06 角色 — 维持设计稿
- **决策**：P06 与 P05 共用题型组件，参数化控制题数（P05=1题/P06=2~3题）
- **理由**：孩子视觉上看不出容器和具体题型的区别，开发上一套组件复用
- **PRD 已更新**：删除"P06 = 小组练习容器"独立定义

### ✅ #2：P07 错位 — 改设计稿
- **决策**：旧 P07（过关庆祝 DWBnB）降级为全局组件；新画 P07 单元测验页
- **理由**：单元测验是 MVP 核心 checkpoint，缺它 P08 结果页无来源
- **设计稿动作**：DWBnB 改名"过关庆祝-组件"；新增 P07-单元测验（基于 P06 视觉 + 顶部进度条）

### ✅ #3：P10 角色 — 维持设计稿
- **决策**：P10 = 小星宝库（成就收藏），分享能力作为子入口
- **理由**：儿童视角是收集成就，不是发朋友圈；分享是大人逻辑、降级为子能力
- **PRD 已更新**：P10 由"分享卡片"改为"小星宝库（含分享能力）"

### ✅ #4：P12 操作引导 — 删除独立页
- **决策**：删除 P12 独立页，改为各页首次使用的上下文引导组件
- **理由**：3-6 岁孩子看不懂引导页，必须在按钮旁直接小手指
- **PRD 已更新**：删除 P12，新增"上下文引导"组件定义

## 4. 变更规则（防止流程漂移）

1. **新增页面**：必须先在本文件「页面权威清单」登记编号、角色、必要性，再动设计稿
2. **新增按钮/跳转**：必须先在「按钮去向表」登记去向，再动设计稿
3. **设计稿与本文件不一致**：登记到「偏离清单」，提交决策，禁止"先改了再说"
4. **PRD 与本文件不一致**：以 PRD 为准，更新本文件
5. **每次会话开始**：AI 助手必须先读本文件再动设计稿（CLAUDE.md 已硬约束）
