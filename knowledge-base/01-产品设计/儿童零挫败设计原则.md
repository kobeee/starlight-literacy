# 儿童零挫败设计原则 · 星光识字

> 本文件记录 3-6 岁儿童识字产品的交互反馈设计红线。与 [[反AI设计规范]] 是**并列**的硬约束：一个管视觉审美、一个管情感反馈。任何涉及"对错判断"、"进度展示"、"奖励机制"的页面设计**必须对照本文**。

## 为什么有这份规则

成人向学习产品（Duolingo、扇贝、英语流利说）的反馈设计核心是**通过负反馈施加压力来驱动持续学习**：错一次就扣心/扣积分/掉连续天数，制造损失厌恶。这种机制对成人有效，因为成人有抽象的目标感（雅思 7 分、考研、升职）压着。

**3-6 岁儿童完全不吃这一套**：
- 抽象目标对他们不存在，只有"现在好玩/现在不好玩"
- 负反馈会直接导致放弃 → 家长再也打不开 APP → 付费曲线断崖
- 认知发展研究显示，这个年龄段"挫败感"的耐受阈值只有成人的 1/5 左右

所以**星光识字的所有反馈设计必须反 Duolingo**。下面四条是否决项。

---

## 原则一 · 三态分类取代二元判断

### 规则

**禁止"已学 / 未学"、"已过关 / 未过关"这类二元标签**。所有掌握度展示必须至少有三档：

| 档次 | 视觉 token | 语义 | 场景 |
|---|---|---|---|
| 金星（mastered） | `$starlight-gold` #FFD166 | 已掌握 | 连续 3 次正确 |
| 银月（familiar） | `$silver-moon` #5B8FC7 | 熟悉中 | 至少 1 次正确 |
| 绿芽（learning） | `$green-sprout` #5BA88E | 学习中 | 接触过但未正确 |
| （灰 / 待解锁） | `$placeholder-gray` #B5A898 | 尚未开启 | 还没解锁的内容 |

### 为什么

"绿芽"档本身也是**肯定状态**——只要接触过就被命名为"正在发芽"，而不是"还没学会"。这消解了"失败"的存在：孩子在宝库里看到的都是"朋友"，只是亲密程度不同。

### 落地示例

- **P10 小星宝库**：顶部四档 tab（金 12 / 银 18 / 绿 17 / 灰 0），每个字按当前档次着色
- 禁止出现"错题本"、"薄弱字"、"待复习"这类带负向暗示的分类

---

## 原则二 · 永远给星

### 规则

**每次互动完成都必须给星星**，包括答错的情况。差异只在数量，不在有无：

| 结果 | 星星数 | 文案示例 |
|---|---|---|
| 完全正确 | 3 星 | "太棒了～3 颗星到手！" |
| 正确但慢 | 2 星 | "答对啦，多练几次更快哦～" |
| 选错但修正 | 1 星 | "没关系，再试一次就对啦～" |
| 首次选错 | 1 星 | "答错也有星星，继续加油～" |

### 为什么

**星星是持续性正反馈锚点**。Duolingo 扣心是损失厌恶利用，让孩子害怕继续。星星的累积逻辑是 +n（永远不 -），让孩子期待下一次。关键差别：星星的意义不是"衡量表现"，而是"记录出勤"。

### 落地示例

- **P06 答题页** 底部反馈条："答对拿 3 颗星 · 错了也有星星～" —— 把"错也有星"写进 UI 文案，不是代码里偷偷给，而是明明白白告诉孩子
- **P07 庆祝页**：不展示总分（"你得了 85 分"），只展示大字和"你真棒～"

---

## 原则三 · 卡关降难度保通

### 规则

**同一个字连续答错 2 次后，自动降低难度直到通过**，不允许"卡在这一关"：

```
难度阶梯（答错自动降档）:
  L3 四选一（水 / 火 / 木 / 山）→
  L2 二选一（水 / 火）→
  L1 直接给正确选项单独高亮 + 语音提示"这就是'水'" →
  L0 不算互动，系统替答 + 继续下一题
```

降档过程对孩子**无感**——UI 上看不出"降了级"，题目看起来还是正常的题目，只是选项变少了。家长后台可查降档记录用于难度反馈，但孩子端完全屏蔽。

### 为什么

3-6 岁没有"我正在学"的元认知。一道题卡超过 30 秒，注意力已经转移。如果这时候还在出原题，孩子就会直接退出 APP，之后很长一段时间不肯再打开。必须在孩子意识到"我不会"之前，悄悄把题目换成"我会"的样子。

### 落地示例

- **P06 答题页** 的系统逻辑（非 UI 可见）：卡 30s 或连错 2 次 → 隐去错误选项变二选一 → 再错 → L1 单选高亮
- **P07 庆祝页** 必然会到达：所以每道题最终都是"过关"，从来不存在"失败页"

## 原则四 · 反 Duolingo 负反馈对照表

以下成人产品的常见机制**在星光识字里禁用**：

| ❌ 禁用机制 | Duolingo 等成人产品如此做 | 星光识字对应做法 |
|---|---|---|
| 红叉 / 红色错误提示 | 立刻弹红色"wrong!" | 正确答案自动高亮金色 + 语音"再听一次'水'"，无负面色 |
| 扣心 / 扣血 | 答错扣一颗心，5 颗扣完锁关 | 永远没有"心"这个概念 |
| 连续天数中断警告 | "你的 30 天连击即将断掉！" | 连击展示为成就（"今日学了 5 字～"），断了不提醒 |
| 排行榜 / 段位 | 跟别人比，掉段焦虑 | 只和自己比（金星从 3 涨到 12） |
| 复活券 / 付费续命 | 扣完心得等冷却或付费 | 付费点只在"解锁新单元"，不在"免除惩罚" |
| 黑红警示弹窗 | 深红底 + 白字大感叹号 | 任何弹窗用温暖色（金 / 粉 / 薄荷），不用红 |
| "Your streak is at risk" | 威胁性文案 | 全文案禁止"警告 / 失败 / 错误 / 扣除" |

### 关键洞察

负反馈驱动对成人有效的前提是"用户有抽象目标"。儿童没有抽象目标，只有此刻体感。此刻体感只要有一次"不开心"就可能永久流失用户，负反馈机制 ROI 是负的——获取成本比 Duolingo 高一个量级，不能为了一时的"留存数据好看"损失付费家长的信任。

**付费家长的核心判断标准是**：孩子愿不愿意主动打开 APP。主动打开的前提是"上次用完的记忆是好玩的"。任何单次不好玩的体验都会直接影响 LTV。

## 本原则的执行审计

每次设计评审必须对照以下检查表：

- [ ] 有没有"错 / 败 / 失 / 输"的文字出现？
- [ ] 有没有红色（除装饰性 brand red 外）出现？
- [ ] 每次互动是否都有正向结果（星 / 奖励 / 鼓励文案）？
- [ ] 有没有可能让孩子"卡住"的节点？卡住后有没有降档机制？
- [ ] 是否有与其他孩子比较的维度？（应删除）
- [ ] 进度条 / 分级 / 分类是否有至少 3 档（不是二元）？

## 理论参考（待补具体文献）

- **Vygotsky 最近发展区（ZPD）**：儿童学习在"可达挑战"区间最有效，任务难度必须可以跳一下够得到，负反馈会把实际难度推出 ZPD → 进入挫败放弃区
- **Carol Dweck 成长型思维（Growth Mindset）**：早期定性评价（"你不聪明"/"你错了"）会让孩子形成固定型思维，长期抑制尝试意愿
- **Duolingo 负反馈机制对成人有效但儿童产品需谨慎套用**——具体数据需查证 APP 留存研究后补充
