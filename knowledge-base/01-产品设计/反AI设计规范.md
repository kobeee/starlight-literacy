# 反 AI 设计规范 · 星光识字

> 本规范基于用户 2026-04-18 在 P01 学习地图评审时给出的**两条硬约束**整理，结合 starlight-literacy 项目现状逐条做"是否已踩雷"审计。
> 下次做任何视觉调整前**必须对照本文件**，避免重复踩坑。

## 为什么要这份规范

"AI 生成感"在视觉上有极强的辨识度——紫蓝渐变、霓虹发光、cyan-on-dark、四平八稳对称居中、万物圆角卡片、Inter/Roboto 等默认字、彩虹六色平均分配——一眼就"像 AI 做的"。儿童识字类产品尤其要避免这种廉价感，否则产品气质会被视觉反噬。

以下两条规则是**否决项**，不是建议项。违反即要求返工。

---

## 规则一 · 反 AI 审美硬约束

| # | 规则 | 本质问题 | starlight 现状 |
|---|---|---|---|
| 1.1 | **禁止紫色 / 多彩渐变** | AI 默认美学的头号特征 | 🔴 **重度踩雷** |
| 1.2 | **禁止 glow 效果作为主要视觉** | AI 生图的廉价"magic"堆砌 | 🔴 **重度踩雷** |
| 1.3 | **一个视图最多一个强调色** | 彩虹平铺=信息层级丢失 | 🔴 **重度踩雷** |
| 1.4 | **不用纯黑纯白**，中性色带品牌色调 | 对比度过硬，缺温度 | 🟢 **基本达标** |

### 1.1 禁止紫色/多彩渐变

**现状审计**：
- `DESIGN.md` 定义的主品牌色就是 `dream-lavender #E8E0F0 → sky-powder #D4E8F7`（紫→蓝渐变），直接撞线
- `soft-purple #B5A7D5` 作为品牌强调色 token 存在
- 场景渐变定义：`Dreamland Home`、`Night Celebration`（紫蓝渐变）、`Golden Hour`
- `char-stroke-active` 是 `linear-gradient(135deg, #FF7B9C, #FFD166)` 粉→金彩色渐变

**已修复**：P01 / P02 / P03 / P04 / P05 / P06 / P07 / P08 / P09 / P10 / P11 已完成暖彩迁移，主方向从“紫蓝/粉橙渐变”纠偏为**奶油米底 + 蜂蜜金主强调 + 暖棕文字 + 低饱和花田辅助色**。

**当前结论**：项目已经不再以“清掉所有颜色”为目标，而是以**统一暖光色温**为目标。允许辅助色存在，但禁止回到 AI 默认紫蓝渐变和平均分彩虹。

### 1.2 禁止 glow 效果作为主要视觉

**现状审计**：
- `DESIGN.md` 品牌关键词明确写："Starlight & sparkle as brand signature — scattered stars, twinkling effects, golden highlights"
- 按钮定义里 "Idle animation: subtle breathing glow (shadow pulse)"
- Unit 卡片 Current 态："shadow glow #FFD166, animated sparkle"
- P01 原设计里充斥 pulse/halo/glow 装饰（4/18 已全部删除）

**已修复**：P01 所有光晕元素删除（MjJLm, OG5L1, SZThg, bQwKc, 2193b 等）。

**当前状态**：
- glow 已不再作为主要视觉语言
- 允许保留极轻的金色阴影来表达厚度和奖励感，但禁止 halo 式霓虹光晕
- “星光”保留为品牌记忆点，不再等于“满屏闪烁特效”

### 1.3 一个视图最多一个强调色

**现状审计**：
- `DESIGN.md` 定义的 "Rainbow Candy" 彩虹配色有 **6 个强调色**：coral-pink / mint-green / sky-blue / soft-purple / peach-glow / sunshine-yellow
- 每个都有具体用途分配（CTA / success / secondary / premium / warm / celebration），意味着每个页面都会同时使用 3-5 个强调色
- 这是"AI slop 色板平均分配"的典型表现

**已修复**：P01 只保留 `starlight-gold #FFC947` 作为唯一强调色，完成/锁定状态改走**灰度差异**（Unit01 暖灰 #B8B0B0，Unit03 冷灰 #D4D0CC）。

**当前执行方式**：默认页面主强调色统一为 `#FFC947` 蜂蜜金，其余花田色只用于信息分组、字卡分类或家长端轻辅助。

### 1.4 不用纯黑纯白

**现状审计**：
- `text-primary #3D3D3D` 暖深灰（不是纯黑）✅
- `warm-cream #FFF8F0` 暖米白（不是纯白）✅
- P01 4/18 重建用 `#FAF6F0` 底色 + `#3A2D20` 深棕字体 ✅

**评估**：这一条 starlight 从一开始就做得对，保持即可。

## 规则二 · 反 AI slop 创作框架

| # | 规则 | 本质问题 | starlight 现状 |
|---|---|---|---|
| 2.1 | **Design system 先行，不是做完再贴** | 事后贴规范=规范失效 | 🟡 **部分落地** |
| 2.2 | **禁用 AI 色板**（cyan-on-dark、紫蓝渐变、暗底霓虹） | AI 默认配色家族 | 🔴 **重度踩雷** |
| 2.3 | **不是所有东西都包在卡片里** | 卡片泛滥=视觉扁平 | 🔴 **重度踩雷** |
| 2.4 | **不要对称居中一切** | 对称=无重心=AI 感 | 🟡 **部分踩雷** |
| 2.5 | **字体要有个性**，禁用 Inter/Roboto/Arial | 默认字=无品牌感 | 🔴 **重度踩雷** |
| 2.6 | **色板要主色+锐利强调色**，不平均分配 | 见规则 1.3 | 🔴 **重度踩雷** |
| 2.7 | **灰色文字不放彩色背景上** | 低对比度可读性差 | 🟡 **待审计** |
| 2.8 | **间距要有节奏**，不要全部一样 | 等距=机械感 | 🟡 **部分踩雷** |

### 2.1 Design system 先行

**现状**：starlight 项目有 `DESIGN.md`（465 行）和 `.impeccable.md`，属于"规范先行"结构。但：
- 规范内容本身与规则 1/2 冲突（见 1.1/1.3/2.5）
- 规范是 4/14 初创时写的，4/18 P01 重建时发现规范本身要改
- **本文件 = 对 DESIGN.md 的 meta 级约束**，以后做设计先看本文件，再看 DESIGN.md

**已完成**：`DESIGN.md` 与 `.impeccable.md` 已重构为田园暖彩体系，后续视觉稿以新设计系统为准。

### 2.2 禁用 AI 色板

**现状审计**：
- `DESIGN.md` 的主品牌色 `dream-lavender #E8E0F0 → sky-powder #D4E8F7` = 典型紫蓝渐变 AI 色板
- P08 测验结果页场景 `Night Celebration: #2A4494 → #4B6CB7` = 暗底蓝紫 = 接近 cyan-on-dark 亲戚
- P09 付费页 `Golden Hour: #FFE8D6 → #FFD4B8` 偏 AI 套路渐变

**已修复**：全项目主视觉已从旧 AI 色板迁出，当前权威色板为奶油米 / 蜂蜜金 / 暖棕 / 花田低饱和辅助色。

### 2.3 不是所有东西都包在卡片里

**现状审计**：
- `DESIGN.md` 定义 5 种 card token：card-white / card-cream / card-blush / card-mint / surface-light
- 几乎每个信息都包在 pill 或圆角矩形卡片里
- P01 原设计的所有标签都是 pill 背景（QfLNe, cevef, PaVjA 等 8 个 pill）

**已修复**：P01 所有文字标签去卡片化，`bigNum`、`u2Name`、`u1Name` 等纯文字直接定位，不套 pill。

**当前状态**：核心流程页已经明显减少“文字也套卡片”的问题。后续开发阶段继续按这个原则收紧即可：标签、数字、状态说明优先裸露；只有可点击功能块和明确内容分组才上卡片。

### 2.4 不要对称居中一切

**现状**：
- 原 P01 所有元素沿中轴对称排列（典型 AI 布局）
- **已修复（P01）**：Unit02 节点左置 120×120，U1/U3 右置 96/72；Unit02 标签右置、Unit01 标签左置，主动破对称
- 主视觉 Unit02 120px，U1 96px，U3 72px 非等大，y 间距 258/204 非等距，形成节奏

**未修复**：其他 7 页的视觉重心、信息分布需要逐页检查是否破了对称。

### 2.5 字体要有个性

**现状审计**：
- `DESIGN.md` 明确写 "Chinese: PingFang SC" —— PingFang SC 是中文的 Arial/Helvetica 等价物，**毫无个性**
- "Pinyin / English: Inter" —— Inter 是被明确点名禁用的 AI 默认字
- 实际开发阶段如果不换，产品气质直接垮掉

**已尝试**：P01 4/18 重建试过 `ZCOOL KuaiLe`（站酷快乐体）和汉仪乐喵体，但 Pencil MCP 渲染器回退到 PingFang SC。**Pencil 阶段无法真的验证字体个性**。

**待做（开发阶段）**：
- 中文：`ZCOOL KuaiLe`（站酷快乐体，免费）或 `汉仪乐喵体`（商用收费但儿童产品气质最佳）或 `方正字迹-童体硬笔`
- 英文/拼音：`Fredoka`（圆胖有童趣）或 `Baloo 2`，**绝不用** Inter/Roboto/Arial/SF Pro
- 引入时记得 `font-display: swap` 并处理 FOUC

### 2.6 色板主色+锐利强调色

参见规则 1.3，同一规则从"色板结构"视角重述。

**原则**：一个页面里应该有**70-80% 中性色（米、暖白、浅灰）+ 1 个主强调色（蜂蜜金）+ 少量低饱和辅助色**。不是“彩虹六色各占 15%”。

**当前示范**：P02/P03/P05/P07/P10 已验证“统一暖光下的多色辅助”比“全部单色”更符合儿童产品气质。

### 2.7 灰色文字不放彩色背景上

**现状**：**待全页面审计**。

**典型问题**：灰色小字（如 hint、caption）放在彩色渐变卡片上会低对比度可读性差。

**检查方法**：用 `search_all_unique_properties` 收集所有 text 的 `fill` 和父节点 `fill`，交叉检查。

### 2.8 间距要有节奏

**现状**：`DESIGN.md` 的 typography hierarchy 行高都是 1.3/1.4/1.5，padding 大量 24px，gap 常用 16px，**整体偏等距**。

**已修复（P01）**：y 间距非等距（258/204），节点非等大（72/120/96），打破等距。

**未修复**：其他页面的 padding/gap 体系需要引入大小梯度（如 4/8/16/32/64 的 2 倍数序列）而非全 16。
