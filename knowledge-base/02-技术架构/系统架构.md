## 前端架构

### 分层设计

```
表现层（Pages + Components）
    │
    ├── 页面层：12 个页面，各自独立
    ├── 组件层：可复用 UI 组件
    │     ├── CharCard       # 汉字卡片
    │     ├── WritingCanvas  # 书写画布
    │     ├── DragBubble     # 拖拽气泡
    │     ├── AudioPlayer    # 音频播放
    │     ├── ProgressBar    # 进度条
    │     ├── StarRating     # 星级评价
    │     ├── ParticleEffect # 粒子特效
    │     └── ...
    │
业务逻辑层（Utils + Services）
    │
    ├── LearningService    # 学习流程控制
    ├── QuizService        # 题目生成与判定
    ├── ProgressService    # 进度管理
    ├── AudioService       # 音频管理
    ├── CacheService       # 缓存策略
    ├── PaymentService     # 支付处理
    └── ShareService       # 分享功能
    │
数据层（Storage + Cloud）
    │
    ├── LocalStorage       # 本地持久化
    ├── CloudSync          # 云端同步
    └── ResourceLoader     # 资源加载与缓存
```

### Skyline 特性使用规划

| 特性 | 应用场景 |
|------|---------|
| Worklet 动画 | 书写页逐笔反馈、气泡弹性动画 |
| 手势系统 | 拖拽匹配题型、书写笔画检测 |
| 共享元素 | 学习地图 → 单元入口转场 |
| 自定义路由 | 认→写→练流程转场动画 |
| scroll-view 增强 | 学习地图纵向卷轴 |

## 后端架构（云开发）

### 云函数设计

无状态函数，按业务域划分：

- **user/**：注册、登录、信息更新
- **learning/**：进度同步、数据上报
- **content/**：单元/字库数据查询
- **payment/**：下单、回调、权益校验
- **share/**：卡片数据生成

## 离线与缓存架构

```
资源请求 → 本地缓存命中？
           ├── 是 → 直接返回
           └── 否 → CDN 请求
                    ├── 成功 → 写入缓存 → 返回
                    └── 失败 → 占位资源降级
```

- 缓存策略：当前单元 + 下一单元预加载
- 存储上限：同时缓存 3 个单元（~30MB）
- 进度数据：始终本地持久化，联网时同步云端
