# 2026-04-24 P01 长卷与 Flow 误删事故记录

## 事故摘要

本次 Codex 会话在处理“清理旧非正式页面”时，错误地把以下画板当成可删除旧稿处理：

- `K8mKq`：`P01-v1-备份(原稿)`
- `pi991`：`P01-v2-田园长卷(完整画布)`
- `Nzevx`：`P01-可视区(当前关居中)`

用户明确指出：

- `pi991` 不是废稿，而是首页长轴画卷主体
- 相关 flow / 长卷视图不应被按“旧稿”清理
- 下一个新 session 需要把这些内容重新画回来

## 直接原因

1. 在没有二次确认的前提下，把“备份 / 预览 / 长卷”误判成“非正式旧页面”。
2. 仅凭顶层画板命名做清理，没有遵守“P01 长卷 / flow / 权威流程板默认保留”的规则。
3. 删除后又对 `design.pen` 文件层做了同步删除，导致本地文件也丢失了这 3 个顶层节点。

## 恢复尝试与结果

已尝试但未成功的恢复来源：

- `git` 历史：仓库当前 `HEAD` 中没有这 3 个节点
- `.codex/worktrees/.../design.pen`：未命中对应节点
- 当前会话 `jsonl`：能确认这 3 个节点曾存在、并能确认名称与坐标，但没有直接拿到完整节点 JSON 供原样恢复

本次会话结论：

- 不继续盲目恢复，避免二次破坏
- 改为在下一个新 session 中按已知记录重新绘制
