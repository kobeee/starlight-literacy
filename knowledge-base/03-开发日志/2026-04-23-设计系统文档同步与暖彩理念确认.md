# 2026-04-23 设计系统文档同步与暖彩理念确认

## 背景

4/23 视觉稿已经基本迁到“田园暖彩”体系，但项目级文档 `DESIGN.md`、`.impeccable.md` 仍停留在早期“梦幻渐变 / 彩虹糖果 / 星空童话”叙事，存在两个风险：

1. 后续 Agent 继续按旧色板出稿，把视觉又带回紫蓝渐变和粉橙 glow。
2. 团队无法明确判断：现在这套暖彩到底是不是更符合品牌关键词。

## 本次同步内容

- 重写 `DESIGN.md`，将权威设计系统从“梦幻马卡龙渐变”切换为“田园暖彩 + 暖阳绘本”。
- 重写 `.impeccable.md`，将高层设计语境同步为奶油米底、蜂蜜金主强调、暖棕主文字、低饱和花田辅助色。
- 更新 `CLAUDE.md`，明确 `DESIGN.md` + `.impeccable.md` 是当前田园暖彩权威源。
- 更新 [[反AI设计规范]]，把“未修复”状态同步到当前进度，避免规范文本落后于真实设计稿。
- 更新产品设计索引与项目简介，移除“星空渐变 / 纯净极简”这类已不准确表述。

## 判断：当前暖彩体系是否符合品牌关键词

### 结论

**是，且比旧“梦幻渐变/彩虹糖果”体系更贴近产品理念。**

### 为什么更符合

#### 1. 更符合“天真 / 无邪”

- 旧体系的问题不是“不好看”，而是带有明显“被设计出来的甜”和 AI 默认浪漫感，像滤镜，不像儿童真实世界。
- 现在的奶油米、暖棕、蜂蜜金、花田辅助色更像自然环境里的颜色，被太阳晒过，稚气更真实。

#### 2. 更符合“快乐 / 幸福 / 无忧无虑”

- 快乐不等于高饱和和满屏庆祝。
- 对 3~6 岁孩子来说，真正让人放松的是轻松、明亮、鼓励感明确，而不是被过强视觉刺激包围。
- 暖彩体系保留了色彩丰富度，但降低攻击性，所以更有“安心玩”的感觉。

#### 3. 更符合“有趣 / 好玩”

- 好玩来自可点击感、奖励感、插画感、节奏变化，不来自 glow 和多彩渐变本身。
- 当前系统里，CTA、字卡、题面卡、花田分组已经能提供足够的趣味和探索感。
- 尤其 P02 / P03 / P05 / P07 / P10 已经证明：统一暖光下的多色辅助，比“彩虹平均分配”更耐看，也不无聊。

#### 4. 更符合家长侧的“愿意付费”

- 家长不会为“廉价梦幻感”买单，但会为“温暖、精致、看起来护眼且有审美”的体验买单。
- 暖彩体系一方面让孩子愿意点，另一方面让家长觉得这是认真做过的产品，而不是 AI 批量套壳。

## 也要注意的边界

当前体系是对的，但有两个边界要守住：

1. **不能继续去饱和到“咖啡店菜单”**
   只保留米色和深棕会失去儿童产品该有的快乐感。

2. **不能回弹到“糖果渐变嘉年华”**
   一旦重新启用紫蓝/粉橙大渐变、平均分彩色和 glow，品牌就会重新变得廉价。

正确方向不是“更少颜色”，而是**更多被暖阳统一过的颜色**。

## 当前建议

- 设计稿层面，当前暖彩体系可视为正式方向。
- 开发阶段优先把新 token 落到代码中，不再参考旧“梦幻渐变”描述。
- 后续只需要继续处理少量素材级遗留，例如 P08 烟花插画、P04 手势图轻量化。
