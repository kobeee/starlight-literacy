# 2026-04-18 学习地图 P01 蜿蜒小路尝试与失败

## 任务目标

用户对 P01 学习地图的"路径"反复不满意，从 v2 的紫色圆点到 v3 的米色踏脚石都被否。明确要求："像 Duolingo 那样的 S 型蜿蜒路径，节点沿着弯曲的小路分布，路径本身要有'走小路'的感觉，不是几个圆点连一条直线。用 SVG 曲线画一条蜿蜒的小路，节点放在路上。"

## 本轮做的事情

### 前置：地图整体重建（对照用户两条规则）
1. **规则一（反 AI 硬约束）**：禁紫/渐变、禁 glow、一个视图单强调色、禁纯黑白
2. **规则二（反 AI slop）**：禁全卡片、禁对称居中、字体有个性、主色+锐利强调色、空间有节奏

按规则重建得到 8/10 版本：
- 底色改单色 #FAF6F0
- 删掉所有 glow/光晕
- 金色 #FFC947 作为唯一强调色（Unit02），Unit01 暖灰、Unit03 冷灰
- 节点非等大：72/120/96；y 间距非等距：258/204
- 标签全去卡片化，位置破对称（Unit02 标签右置、Unit01 标签左置）
- 地平线构图：上半米色干净，下半 AI 插画作地面

### S 型蜿蜒小路的失败

用户明确要求 SVG 曲线画蜿蜒小路。技术尝试路径：

1. **直接 path 节点（type:"path"）absolute 定位**
   ```javascript
   I("Qm8Jg", {type:"path", layoutPosition:"absolute",
     x:0, y:0, width:390, height:844,
     viewBox:[0,0,390,844],
     geometry:"M 268 648 C 280 540, 72 478, 120 390 C 148 316, 340 276, 260 186",
     stroke:{fill:"#E8DCC5", thickness:16, cap:"round", align:"center"}})
   ```
   **结果**：节点写入文件成功（batch_get 读回 geometry 完整），但 `get_screenshot` 完全不渲染。

2. **极端参数测试**：改成纯红 30px 粗线，仍不渲染。

3. **wrapper frame 包裹**：用 `layout:"none"` 的 frame 套一层 path，仍不渲染。

4. **最终妥协：贝塞尔采样点 + 踏脚石椭圆**
   手算两段三次贝塞尔的控制点，沿 t=0.2~0.8 采样 12 个点，每点放 14×14 深棕 + 奶油描边椭圆。

### 用户最终反馈

> "你看看那几个小圆圈，那是路吗？搞笑的吗？"

—— 承认失败。12 颗椭圆再密也是"点阵"不是"路"。视觉上不等效于连续曲线。

## 失败复盘

### 技术层
**Pencil MCP 当前渲染器对 `type:"path"` 节点存在无法渲染的 bug**（至少在 MCP 截图与导出路径），是本轮无法交付的硬门槛。不是我写错 schema，而是工具本身限制。

### 判断层（更重要）
- **看到 path 不渲染就应该停下来说明限制**，而不是用"椭圆踏脚石"硬凑一个"看起来像路"的妥协方案
- 用户的核心诉求是**连续曲线小路**，密集圆点在视觉本质上是另一种东西，再多也不是路
- 早点承认能力边界反而更专业，继续凑只会消耗用户耐心

### 审美层
- 即使是"踏脚石"方案，12 颗分散在大屏上中间距离 40-100px，也不构成视觉连续性
- Duolingo 的"路"有两个视觉成分：底层连续曲线（路面）+ 上层节点（关卡）。只靠节点撑不起"路"的感觉

## 下次再续的方向

用户说"下次再继续吧"。方向备选：

1. **用 icon_font 里的路径图标**：Material Symbols 可能有 `waves`、`timeline` 等可以借用
2. **走真代码路线**：在 React/小程序开发阶段用真 SVG，Pencil 只做静态节点位置参考
3. **改变设计语言**：彻底放弃"路径"叙事，换成别的视觉比喻（如浮岛、阶梯）
4. **等 Pencil MCP 修复 path 渲染**或换其他工具（Figma Plugin？）

## 结论

本轮交付 = 重建得到 8/10 的地图底盘 + 失败的蜿蜒小路尝试。**P01 视觉定稿暂停**，P01 作为未完成项进入下次。
