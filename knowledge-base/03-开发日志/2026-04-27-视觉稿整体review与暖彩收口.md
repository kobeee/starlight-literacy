## 遇到的问题
- Pencil CLI 打开时仍会尝试拉取部分 Google/Noto 字体并超时；本次编辑器进入 Ready，`get_editor_state` 和 `save()` 都成功，因此未阻断本地保存。
- 之前出现过 `save()` 输出 0B 文件的事故，本次继续执行“只信校验后的 `/tmp/*.pen`”流程，避免直接覆盖正式稿。
- Pencil interactive 对脚本节点存在缓存：删除临时遮盖层后，必须重启 headless 再导出验证，否则可能看到旧脚本渲染结果。
- P01 底部小房子比例与画面不协调，已按“高效一点”原则直接移除，不再继续生成或修融合度。

## 关键决策
- 视觉基线继续以田园暖彩为准，不回到冷色科技感、紫蓝渐变或高饱和奖励风格。
- 结果页和测验页不展示会制造挫败感的考试指标，不使用“重考/再试一次/正确率/用时”等语言作为主反馈。
- 任何图片生成或视觉 review 都不把原始大图塞进请求；只用纯文本生成、本地资产和压缩缩略图复核。
- Pencil 保存必须走安全流程：备份 `design.pen` → `pencil interactive -i design.pen -o /tmp/xxx.pen` → 单独 `save()` → 校验非空 / `jq empty` / 关键引用 → 覆盖正式稿。

## 学到的知识
- Pencil CLI 的 `save()` 输出“Saved”不等于文件一定安全；必须用 `test -s`、`jq empty` 和关键引用扫描确认。
- 字体拉取超时和保存失败是两类问题：只要编辑器 Ready 且本地资源已加载，仍可完成结构读取和保存。
- “统一色调”不是单色化；儿童识字产品更适合暖米底 + 蜂蜜金主强调 + 低饱和花田色点缀，而不是整页同一色相。
- 对脚本节点做视觉验收时，优先重启 Pencil headless 后重新 `export_nodes`；不要信同一进程里连续修改脚本后的缓存导出。
