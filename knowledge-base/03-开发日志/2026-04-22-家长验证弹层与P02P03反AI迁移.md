## 家长验证题弹层（W7pdt）

### 设计要点
- **题面**：3 + 5 = ?（一位数加法，正常成人秒答，3-6 岁童难答）
- **键盘**：3×3 数字（1-9）+ 第四行 [退格 / 0 / 确认]
- **配色**：暖米底 #FAF6F0 + 数字键白底 + 退格暖灰 #EFE7DC + 确认金 #FFC947 + CTA 深棕 #3A2D20
- **零彩色 / 零 glow / 零渐变**：完全合规反 AI 规范
- **CTA**："通过验证 + shield_person 图标"

### 踩坑
- **I() 嵌套 frame 静默空白**：第一次想从零 I() 建 modal 卡片，截图全空。改用 C("ws7mn") 复制 P02 整页再 R/U 改造，验证可行。
- **`layoutPosition:"absolute"` 在 vertical layout 子节点上不生效**：banner（fill_container width）内部的子 text 给 absolute 定位无视，仍按 layout center 渲染。
- **`justifyContent:"center"` 在某些 banner 高度 < 子内容高度时不正确**：child 渲染到容器底部并被 clip。修复：把 banner 删掉，让 text 直接做 W7pdt 的 layout child 之一。
- **回滚副作用**：banner 删除指令在第一批 batch 中因 effect:null 校验失败被回滚，连带 6 个绝对定位装饰 icon 也没删，导致 "3+5" 文字被装饰星星压住。第二轮单独清理才干净。

### 终止条件触发
原计划做"答案输入框 + 实时回填"的精细交互，但 banner 内部 absolute 失败 + child clip + 装饰 icon 残留累计 5 轮调试后，决定**收手**：题面静态 "3 + 5 = ?" + 键盘 + CTA 已能完整表达"家长验证"语义，输入区交互留给开发阶段。**符合"不要钻牛角尖"的硬约束**。

## 工具技巧沉淀

### `replace_all_matching_properties` 是核武器
P02 row2-4 共 16 个 cell 用紫色 #EDE8F4 + 16 段绿字 #3EAA8F + 16 段绿拼音 #5BB89E，逐个 U() 要 50+ 操作。改用 `replace_all_matching_properties({parents:["ws7mn"], properties:{...}})` 一次完成 5 种属性映射，省 80% 工作量。**反 AI 大批量颜色迁移首选这个工具，不要手 U()**。

### 题目盒不可强求 vertical center
在 P02 复制后保留的 banner 容器（fill_container width）里，`justifyContent:"center"` + `padding:0` 的子 text 渲染 y 异常（vertical 模式下 child y > 容器高，被 clip）。**两条解法**：
1. 把 child 移出该容器，作为外层 W7pdt 的 layout child
2. 给容器固定 height + 取消 clip，让超出部分自然显示（但仍位置不对）

实战选了第 1 条 + 删掉 banner 容器。
