# 2026-04-15 UI 设计稿并行实验

## 任务背景
使用三个 AI Agent（Cursor/Opus、Claude Code、Codex）并行创建星光识字的 UI 设计稿，对比不同工具的设计能力。

## 工作流程

### 1. 准备阶段
- 基于 [awesome-design-md](https://github.com/VoltAgent/awesome-design-md) 格式创建了项目专属 `DESIGN.md` 设计系统文档
- 定义了完整的色彩体系、字体层级、组件规范、页面设计说明
- 创建了 3 个 git worktree 做隔离：
  - `design/cursor-opus` — Cursor IDE + Pencil MCP
  - `design/claude-code` — Claude Code CLI
  - `design/codex` — Codex CLI

### 2. 隔离策略
- 每个 Agent 在自己的 worktree 中工作，有独立的 .pen 文件
- 只有 Cursor/Opus 使用 Pencil MCP 交互式设计
- Claude Code 和 Codex 通过 CLI 直接写 .pen JSON 文件
- 避免了同时操作同一个 .pen 文件的冲突风险

## 经验总结

### Pencil MCP 使用心得
1. **icon_font 不支持 fontSize** — 只用 width/height 控制尺寸
2. **AI 图片生成可能超时** — G() 操作超过 60s 会 timeout，用渐变色 placeholder 替代
3. **placeholder 工作流** — 创建 frame 时先设 `placeholder:true`，完成后设 false
4. **截图分辨率低** — get_screenshot 返回缩略图，小文本看不清，需配合 snapshot_layout 验证
5. **flexbox 注意事项** — `fill_container` 只在父元素有 layout 时生效

### DESIGN.md 作为设计桥梁
- DESIGN.md 格式非常适合作为 AI Agent 之间的设计共识文档
- 定义了颜色、字号、间距等可量化规范后，不同 Agent 产出的设计一致性显著提高
- 推荐使用 `getdesign` CLI 工具快速获取参考设计系统

### Git Worktree 并行工作
- Git worktree 非常适合这种并行实验场景
- 每个 worktree 共享 .git 但有独立的工作目录
- 不同 Agent 可以在各自的 branch 上独立提交
