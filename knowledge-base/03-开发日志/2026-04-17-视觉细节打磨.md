### 二、文字对比度提升三件套

对需要强化的标题/数据文字，统一使用"三件套"：

```javascript
U("titleNode", {
  fill: {
    type: "gradient", gradientType: "linear", rotation: 180,
    colors: [
      {color: "#FFFFFF", position: 0},
      {color: "#FFF2C7", position: 0.5},
      {color: "#FFD166", position: 1}
    ]
  },
  fontSize: 44, fontWeight: "800",
  effect: {type: "shadow", offset: {x: 0, y: 3}, blur: 12, color: "#1A2F6080", shadowType: "outer"}
})
```

要点：
- **渐变文字填充**：text 节点的 `fill` 属性直接支持 gradient 对象
- **双重加强**：字重从 700 拔到 800，字号拔 20%
- **投影定色**：深色底用半透明深蓝 `#1A2F6080`，浅色底用对应主色的 30% 透明度

### 三、数据并列差异化套路

P08 测验结果 3 个数据（8/8、100%、3分）、P09 3 张价格卡、P11 3 张统计卡，统一采用"三色区分"：

| 数据类型 | 主色 | 背景渐变 | 描边 |
|---|---|---|---|
| 成绩/累计 | starlight-gold `#FFD166` | gold-26→gold-10 | gold-55 |
| 正确率/完成 | mint-green `#7ECFB8` | mint-26→mint-10 | mint-55 |
| 时长/次数 | sky-powder-deep `#8BB8E8` | blue-26→blue-10 | blue-55 |

- 数字同色系渐变（浅→深，180°）
- 标签（"答对"/"正确率"/"用时"）采用与数字同色但亮度+30%
- 每张卡独立 `cornerRadius: 14` + `padding: [14, 10]`

### 四、孤立装饰元素聚落化

**问题**：P08 底部右下角一颗 `star` 孤立在 (350, 700)，左侧 (20, 450) 一颗也孤立。

**方案**：以孤立元素为锚点，四周补 3-4 个互补图标形成聚落。

```
右下聚落（锚点 UM4NL）:
  UM4NL star (336,688)  — 锚点，放大到 24×24，金色 75% 透明
  flare     (366,716)  — 伴星，白色 55%
  star      (310,725)  — 伴星，金色 60%
  favorite  (352,740)  — 点缀心，珊瑚粉 50%

左下聚落（锚点 iLgQJ）:
  iLgQJ star (28,660)   — 锚点
  flare     (48,688)   — 伴星
  star      (18,720)   — 伴星
  favorite  (60,735)   — 点缀心

中段节奏点：
  (340,380) flare 10px 白 30%
  (25,340)  star  12px 金 40%
  (360,480) circle 8px 白 20%
  (15,550)  circle 6px 白 18%
```

规则总结：
1. 聚落至少 3 个元素，密度梯度（大-中-小）
2. 颜色/形状混搭（星 + 心 + 闪光），避免同质
3. 透明度从锚点向外递减（75% → 50% → 30%）
4. 中段加低透明"节奏点"让视觉从上到下连续

## 关键发现

### 1. Pencil MCP flex 容器 +50px y 偏移 bug
在 flex-layout 容器内用 `I()` 插入新子节点时，子节点自动带 +50px y 偏移，导致渲染位置超出父 frame 裁剪区。

**复现**：P01 bottomNav 设置 y=756，实际渲染到 y=806；Streak 打卡条子元素 y=64/84（父高 69px），完全被裁。

**规避方案**：
- 不要向 flex 容器 `I()` 新子节点
- 只用 `U()` 修改已有节点
- 必须新增装饰时，`I()` 到顶层 frame 并用 `layoutPosition: "absolute"`（仍有 +50 偏移，但可手动补偿）

### 2. Material Symbols Rounded 图标黑名单（补充）
前期已知 `auto_awesome`、`emoji_events` 不存在。今天验证：

| ❌ 不存在 | ✅ 可用替代 |
|---|---|
| auto_awesome | flare（闪光）/ stars |
| emoji_events | workspace_premium / military_tech |
| auto_fix_high | brush |
| auto_graph | trending_up |

**使用前验证**：批量设计前用 `batch_get({patterns:[{type:"icon_font",name:"xxx"}]})` 搜确认，或先小批量试跑发现错误反馈再替换。

### 3. 文本渐变填充
text 节点的 `fill` 属性完整支持 `gradient` 对象（不只是纯色或变量引用）。这是解决纯色文字在彩底背景上对比不足的最优解。

```javascript
fill: {
  type: "gradient",
  gradientType: "linear",
  rotation: 180,  // 0=左→右, 90=上→下, 180=垂直渐变
  colors: [{color: "#XXX", position: 0}, ...]
}
```

### 4. 批量操作规模控制
单次 `batch_design` 控制在 ≤25 操作，过多会触发潜在问题检测提示且失败回滚全部。今天切实采用"每页 1-2 个 batch"节奏，稳定推进。

## 方法论沉淀

### 视觉单调 → 丰富的通用策略
1. **文字层面**：纯色 fill 改渐变 fill，加 shadow effect，字重 +1 档
2. **卡片层面**：差异化渐变背景 + 彩色描边 + 对应主色阴影
3. **按钮层面**：纯色 → 3 色渐变（浅→深），加白色内描边 `#FFFFFF33`
4. **装饰层面**：孤立元素 → 3+ 元素聚落，透明度递减，形状/颜色混搭
5. **间距层面**：padding/cornerRadius 加 10-30%，gap 可收紧但视觉更透气
