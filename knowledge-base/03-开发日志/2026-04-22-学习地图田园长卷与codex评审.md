1. **回归田园风格，弃"三球+12椭圆"**：04-18 那版"踏脚石点阵"被用户否，回到长卷水彩 + 沿路节点。

## codex 评审结果（8 条建议，按严重度）

> 用 `cat prompt.txt | codex exec --skip-git-repo-check -i 01-bg-only.png -` 跑出，原始输出未单独存档（review.txt 未生成），以下为本 session 实时整理后的版本，下 session 第一件事可重跑 codex 拿到原文。

🔴 **必改（下 session 优先）**

- **#2 当前关突出度不够**：4 号金球与已通关的灰球差异不足，需放大 1.5×，加强结构包裹（木牌 / 台座 / 小旗），让"我现在该点这里"在 200ms 内被孩子识别。
- **#4 8 个单元一次全暴露认知过载**：3-6 岁孩子对未解锁的 4 个浅灰球+锁会产生压迫。建议未解锁单元只露 1 个弱提示（最近的下一关），其余 3 个用云雾遮罩，挡住即可。
- **#7 顶栏 47/1300 数字属于家长视角**：孩子看不懂分数比，且 1300 这个数字本身也是营销数。移到 P11 家长中心，顶栏只保留 Logo + 宝库 + 家长入口。

🟡 **可改（视优先级）**

- **#3 三态形状区分弱**：已通关/当前/未解锁全是圆球，仅靠颜色区分。建议未解锁改为半透明云团形状，已通关改为带星徽的圆，让形状本身就传递状态。
- **#5 标签压在花草上影响画面**：白色圆角标签全压在路右侧，破坏水彩的呼吸感。建议沿路外侧（左右交替）放置，或用半透明描边而非实白底。

🟢 **远期评估（架构级）**

- **#1 信息架构纵向单列偏单调**：未来 1800 字需要展开到 4-5 个板块，纯纵向卷轴会越来越长，可考虑"主路 + 分叉小岛"。
- **#6 顶部云雾"更多旅程"对孩子不直观**：可换成"小气球飘起" / "门洞" 等更具象的"未来入口"隐喻。
- **#8 小程序性能**：单图 1800 高的 PNG 在低端机上滚动可能掉帧，开发阶段需评估切片或瓦片化方案。
