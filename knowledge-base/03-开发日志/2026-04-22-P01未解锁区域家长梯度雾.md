## 四轮迭代复盘

### 轮 1：12 个棉花糖云椭圆（失败）
- 方案：每个未解锁单元头顶 4 个白椭圆（main/left/right/top）+ 描边
- 问题：实色+硬边+三团重复，像贴纸糊在水彩上，风格完全两套语言
- 技术踩坑：`I()` 新建 ellipse 带 effect（blur/shadow）**全部静默失败**（不止 frame，ellipse 也中招）→ 改用 `C()` 复制 u5Main 才渲染

### 轮 2：云朵柔化（仍假）
- 方案：fill 改 `#FFFFFF80`、stroke 去掉、加 blur radius 6
- 问题：是柔了，但**病根不是云雾**，是"4 个一样的灰球+锁"本身就是 UI 贴纸。柔化云只是给贴纸蒙一层雾
- 用户反馈："有点假，且全部都一样的形状，太单调"

### 轮 3：全删（过拟合）
- 方案：删掉 u6/u7/u8 所有节点 + 所有云雾椭圆，u5 改虚线圈+星
- 视觉效果：极好，水彩完全透出来
- 问题：**丢掉了家长视角的"前路感"**。P01 是选关入口，家长需要看到"总共多少关 / 还剩多少"来感知产品价值

### 轮 4：恢复单元 + 单块梯度雾（采纳）
- 方案：
  - u6/u7/u8 灰球+锁+标签全部恢复
  - 用**一块 390×520 的渐变矩形**（从上到下 F2 → B3 → 33 → 00 四段梯度）盖住中上部
  - 雾带起点 y=340，视窗底部几乎无雾，越往上越浓
- 效果：
  - **孩子视窗（Nzevx）**：u4/u5 清晰，雾只在顶部留一道淡边暗示"远方"
  - **家长全景（pi991）**：往上滚，u6/u7/u8 渐次浮现越来越朦胧，能数清规模但不压迫

## 核心认知

### 双受众冲突的解法不是"折中"，是"分层"
- 孩子和家长在 P01 上诉求冲突：孩子要"只看下一步"，家长要"看全貌"
- 折中方案（半透明云雾遮挡）两边都不满意
- **分层方案**（视窗层 vs 滚动层用梯度自然切分）两边都满足

### 水彩风 UI 里，"统一渐变"胜过"分组元素"
- 12 个云椭圆无论怎么柔化都是"UI 元素"
- 一整块线性渐变矩形反而像"晨雾"本身，不被识别为 UI

### 规模感是产品价值信号，不能无脑删
- "前路漫漫"的视觉规模感确实主要是给家长看的
- 但 P01 就是决策入口，家长在这里判断"值不值得给孩子用"
- 轮 3 的删除方案忽略了这个业务语境

## Pencil 技术沉淀

### I() + effect 静默失败（补强既有 memory）
- 已知 `I()` 新建 frame 带 effect 会静默空白
- 本次确认 **`I()` 新建 ellipse 带 effect（blur/shadow）同样会静默空白**
- 可靠方案：`C()` 从已渲染节点复制，再 `U()` 改属性
- 本次 24 个云椭圆走 `C("e7ITd"/"czv12", ...)` 复制 u5Main 才全部渲染

### U() 不能把 stroke 置空
- `U(id,{stroke:null})` 报错 "expected object, got null"
- 解法：`U(id,{stroke:{thickness:0}})`

### Effect blur 属性名是 radius
- schema 是 `{type:"blur", radius: number}`
- 不是 `blur: number`（第一次写错被 schema 校验拦截）
