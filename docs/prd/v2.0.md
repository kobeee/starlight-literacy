【上层：产品理念与设计原则】
====================================================================

一、我们在做什么
────────────────

一款基于微信小程序的儿童识字工具。
对标洪恩识字，最终目标覆盖 1800 个常用汉字。
但我们不会一次性设计 1800 个字——而是一个单元一个单元地打磨、
上线、收集反馈、迭代，让产品随着真实用户的使用不断进化。


二、不变的品牌调性
──────────────────

1. 纯粹
   没有花哨游戏、宠物养成、社交攀比。
   孩子打开就学字，学完就走。

2. 极美
   每一张插画、每一个界面都追求顶级审美。
   AI 是降维赋能的工具，不是偷懒的借口——产出质量必须过人眼关。

3. 极致性价比
   洪恩 1/10 的价格，买断制不套路。
   家长花的每一分钱都能看到明确的内容回报。

4. 护眼防沉迷
   这是写在品牌基因里的底线，不是可选配置。
   （具体规则见下方"家长侧功能"）


三、核心学习模型："认 → 写 → 练"三步法
────────────────────────────────────────

这是产品的骨架，所有单元都遵循这个基本闭环。
但三步法的具体交互形式不是铁板一块——随着字库难度变化和用户
反馈积累，每一步的呈现方式都可能演进。

"认"：建立 字形 ↔ 字音 ↔ 字义 的直觉关联
"写"：通过笔顺描红强化字形的肌肉记忆
"练"：通过互动题目反复巩固，确认真正记住

设计红线（所有单元必须遵守）：
  - 每个字必须完整走完三步才算"学过"
  - 单步交互时长控制在 10 秒以内，低幼儿童注意力极短
  - 每一步都必须有即时的视听反馈（正确 / 错误 / 完成）
  - 视觉信息密度极低，一屏只做一件事


四、单元制与渐进交付
────────────────────

1. 单元是产品的最小交付粒度
   - 每个单元包含一组汉字。
   - 每个单元的字数不固定——根据汉字之间的语义关联、对比关系、
     难度搭配自然分组，可能是 10 字、15 字、20 字不等。
     先把字分好，组多大就多大，不削足适履。
   - 每个单元是一次完整的"设计→开发→上线→反馈"小闭环。
   - 单元之间允许存在玩法差异——后续单元可以引入新的题型、
     新的视觉风格、新的交互方式。

2. 单元内部节奏
   - 单元内的字按 3~5 字一小组拆分（具体视单元总字数而定）。
   - 每组：逐字走完"认→写→练"→ 该组小练习。
   - 全部小组完成后：单元综合测验 → 星级评价（1~3 星）。
   - 通过测验解锁下一单元。

3. 字库规划方向（非锁定，随迭代调整）
   - 长期目标 1800 字，对标部编版小学语文及《现代汉语常用字表》。
   - 早期优先选择：象形字、高频生活字、笔画简单的字。
   - 具体哪些字入哪个单元、每个单元多少字，在每个单元设计时
     再决定——不提前锁死。

4. 学习地图
   - 纵向卷轴路径，星空主题（呼应品牌"星光识字"）。
   - 已完成单元亮起星星，当前单元高亮跳动，未解锁单元灰色。
   - 地图长度随实际上线的单元数动态增长。


五、题型池（持续扩充）
────────────────────

以下是当前规划的题型。首版不需要全部实现——第一个单元只选用
其中最核心的 2~3 种，后续单元根据需要逐步引入新题型。

A. 拖拽匹配（核心题型）
   展示插画，下方 3 个汉字气泡，拖拽正确汉字到插画上。
   拖对 → 爆炸消散 + 撒花；拖错 → 弹簧回弹。

B. 听音选字
   播放语音，展示 3~4 个汉字卡片，点选正确的。
   强化"音→形"关联。

C. 看图选字
   展示插画，下方 3~4 个汉字选项，点选。
   纯点击，快节奏。

D. 字形拼合（适合合体字，后续单元引入）
   汉字拆成 2~3 个部件，拖拽拼合。
   帮助理解汉字结构。

E. 找不同（适合易混淆字，后续单元引入）
   展示 4 个汉字，找出混入的形近干扰字。
   强化字形辨析。

后续可能新增的题型方向（待验证）：
   - 连线题（字与图/字与拼音连线）
   - 填空题（句子中选字填入）
   - 语境选择（简单场景图+选字）


六、家长侧功能
────────────

这些功能从第一版就需要，因为家长是付费决策者。

1. 学习进度
   - 今日：学了几个新字、练习正确率
   - 累计：已学字数、当前单元
   - 极简卡片式呈现，一屏看完
   - 入口：首页右上角"家长"按钮，数学验证题防误入（如"3+5=?"）

2. 防沉迷与护眼
   - 单次学习上限：默认 15 分钟 → 弹出全屏"休息一下吧"→
     锁定 10 分钟
   - 每日上限：默认 3 次/天（共约 45 分钟）
   - 护眼色温：学习界面底色暖白（#FFF8F0），避免纯白刺眼
   - 家长可在设置中调整时长参数

3. 分享炫耀
   - 触发时机：完成单元 / 达成里程碑（如累计 50、100 字）
   - 生成精美分享卡片：孩子昵称 + 掌握字数 + 星星 + 插画背景
     + 小程序码
   - 保存到相册 / 分享至微信聊天或朋友圈
   - 目的：家长晒娃 → 天然裂变 → 免费获客


七、变现模式
──────────

核心策略：阶梯式付费漏斗，极低门槛破冰首单，终身买断兜底。

1. 免费体验
   第一个单元完全免费，让家长和孩子充分感受产品质量。
   这个单元必须做到"让家长想付费继续"的水准。

2. 低价破冰
   - 第二个单元：弹出付费引导 + 优惠券，9.9 元解锁。
   - 9.9 是冲动消费价，目标是让家长"先掏一次钱"，建立付费
     关系。这一步的转化率决定整个商业模型能不能跑通。

3. 后续单元定价
   - 具体价格暂不锁定，根据内容量、用户反馈和付费数据迭代。
   - 方向：按单元单独购买 / 按阶段打包购买，灵活组合。

4. 终身买断
   - 提供一个"一劳永逸"的选项：299 或 399 元解锁全部内容
    （含未来更新）。
   - 这个价位留出了中间定价的充足空间，同时对比洪恩仍有
     性价比优势。
   - 具体定价在有了首批付费用户数据后再最终确定。

设计原则：
  - 每一层付费都要让家长觉得"值了"，绝不做付费后发现是半成品
    的体验。
  - 价格体系保持简洁，家长 3 秒内能看懂自己该付多少钱。


八、复习机制（拓展功能，v1.1 规划）
──────────────────────────────────

非首版刚需，核心流程跑通后第一个迭代上线。

- 简化版遗忘曲线：学后 1天/3天/7天/15天/30天 触发复习
- 复习形式：快速闪卡（展示插画 → 回忆 → 翻转确认 → 记住了/再看看）
- "再看看"的字缩短复习间隔，连续 3 次"记住了"标记为已掌握
- 首页增加"今日复习"入口 + 角标提示


九、离线体验
──────────

- 每次进入小程序时，预加载当前单元 + 下一单元的全部资源
- 学习流程优先读本地缓存，未命中再请求 CDN
- 完全断网时：已缓存单元正常学习；未缓存单元友好提示
- 存储控制：同时最多缓存 3 个单元（约 15~24 MB），自动清理
  最早的已完成单元


十、AI 内容量产工作流
────────────────────

每个单元上线前，执行以下流水线：

1. 数据配置（Claude Opus 4.6）
   生成该单元 20 个字的 JSON 配置：汉字、拼音、组词、释义、
   字源插画生图 Prompt、练习题配置

2. 插画资产（Nano banana pro）
   根据 Prompt 生成 20 张字源插画，人工审核质量

3. 音频资产（AI 语音合成）
   生成 20 条认读语音 + 复用通用音效库

4. 开发集成
   将配置和资产接入小程序，调试交互

【下层：第一个单元的具体设计】
====================================================================

单元编号：Unit-01（启蒙篇·第一单元）
字数：20 字
目标：作为产品的免费体验单元，必须在 20 个字内让家长和孩子
同时被打动——孩子觉得好玩，家长觉得有效且精美。

一、选字原则
──────────

首单元选字标准（比后续单元更严格）：
  - 必须是最高频的生活常用字
  - 优先象形字（字源插画最容易做出视觉冲击力）
  - 笔画数控制在 1~8 画（低幼友好）
  - 字与字之间尽量有可对比性（方便出练习题的干扰项）

20 字初选（待最终确认）：
  一、二、三、大、小、上、下、人、口、手
  日、月、水、火、山、木、土、天、目、耳

分组：
  第 1 组：一 二 三 大 小
  第 2 组：上 下 人 口 手
  第 3 组：日 月 水 火 山
  第 4 组：木 土 天 目 耳


二、"认"模块设计
────────────────

沿用上层定义的基本框架，首单元的特殊考量：

- 这 20 个字全部是象形字或极简字，字源插画要做到"一看就懂"：
  · "火" → 水墨风跳动火焰
  · "山" → 远景连绵山峦，画风大气
  · "目" → 灵动的大眼睛特写
  · "手" → 张开的小手掌，温暖可爱

- 组词选择上偏日常口语：
  · "大" → "大大的"（而非"伟大"）
  · "口" → "口水"或"开口"（而非"人口"）

- 首单元第一个字（"一"）进入时，增加一次简短的操作引导动画，
  教孩子"点一下听声音"和"滑一下写笔画"。


三、"写"模块设计
────────────────

沿用上层框架，首单元特殊考量：

- 前 5 个字（一二三大小）笔画极少，是孩子建立"我能写对"信心
  的关键窗口。容错阈值可以比后续单元更宽松。
- "一"只有 1 笔，写完瞬间成就感爆棚——特效可以做得比其他字
  更夸张一些，让孩子从第一个字就嗨起来。


四、"练"模块设计
────────────────

首单元启用的题型（从题型池中选择 3 种）：

1. 拖拽匹配（题型 A）—— 主力题型
   每组小练习必出 1~2 道。

2. 听音选字（题型 B）
   每组小练习出 1 道。
   首单元的字发音差异大，不容易混淆，适合建立"听→选"的习惯。

3. 看图选字（题型 C）
   单元综合测验中使用。
   快节奏过一遍 20 个字，检验整体掌握度。

暂不启用的题型：
  - 字形拼合（D）：首单元都是独体字，拆不了部件。
  - 找不同（E）：首单元字形差异大，找不同太简单没意义。
  → 这两种题型预计在第 3~5 单元开始出现合体字时引入。

小练习结构（每组 5 字学完后）：
  - 3 道题：2 道拖拽匹配 + 1 道听音选字
  - 干扰项从本组 5 字中选取

单元综合测验（20 字全部学完后）：
  - 8 道题：3 道拖拽匹配 + 2 道听音选字 + 3 道看图选字
  - 干扰项从全单元 20 字中交叉选取
  - 答对 6 道及以上 → 通过（获 1~3 星）
  - 未通过 → 鼓励语"再试一次"，可重考


五、首单元验收标准
──────────────────

上线前必须满足：
  □ 20 个字全部可走完"认→写→练"完整流程
  □ 所有插画经人眼审核，质量达到"家长愿意截图发朋友圈"的水准
  □ 所有音频清晰、音量一致、无杂音
  □ 书写模块笔顺数据正确（对照国家规范）
  □ 练习题的干扰项不会造成歧义
  □ 防沉迷功能正常生效
  □ 家长进度页能正确显示学习数据
  □ 分享卡片可正常生成和分享
  □ 弱网/断网时已缓存内容可正常使用
