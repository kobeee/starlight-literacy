====================================================================
第一部分：产品概述
====================================================================

1.1 一句话定义
    基于微信小程序的"纯净极简、高颜值"儿童识字工具。

1.2 背景与动机
    洪恩识字体验好但价格高（永久包两三百元），部分家长觉得贵。
    我们做一款"平替版"——用 2026 年最新的 AI 工具链把内容生产
    成本打下来，以极致性价比切入价格敏感的家长群体。

1.3 不做什么（明确边界）
    × 不做 3D 动画和重度美术——独立开发者无法对抗洪恩百人产能
    × 不做实时大模型交互——成本高、合规风险大、低幼儿童用不上
    × 不做宠物养成/社交/排行榜——保持纯净，不搞成瘾设计
    × 不做复杂的探索地图——3 岁孩子看不懂

1.4 核心差异化
    ┌────────────┬──────────────┬──────────────────────┐
    │            │  洪恩识字     │  星光识字             │
    ├────────────┼──────────────┼──────────────────────┤
    │ 价格       │ 200~300 元   │ 9.9 起，终身 299~399  │
    │ 内容形式   │ 3D 动画+游戏 │ 极美 2D 插画+极简交互 │
    │ 商业模式   │ 大团队工业化 │ AI 工具链+独立开发者   │
    │ 产品调性   │ 游戏化重度   │ 纯净护眼轻量          │
    │ 字库       │ 1300+        │ 一期 1300 / 远期 1800  │
    └────────────┴──────────────┴──────────────────────┘

第二部分：用户角色与场景
====================================================================

2.1 主要用户

    角色 A：孩子（3~6 岁学龄前儿童）
    - 注意力极短（单次集中 5~15 秒）
    - 需要即时的、夸张的视听正反馈
    - 不识字，依赖图形、颜色、声音理解界面
    - 精细动作能力有限，触控操作需要大按钮、宽容错

    角色 B：家长（25~40 岁，孩子的父母）
    - 付费决策者
    - 关心：孩子学了什么、学了多久、有没有效果
    - 担心：屏幕时间过长、内容是否安全
    - 分享欲：愿意在朋友圈/社群晒孩子的学习成果

2.2 典型使用场景

    场景 1：居家碎片时间
    饭后/睡前，家长把手机给孩子，孩子自主学习 10~15 分钟。
    → 需要：快速启动、防沉迷自动锁定

    场景 2：出行途中
    车上/高铁上，网络不稳定，孩子需要打发时间。
    → 需要：离线可用

    场景 3：家长查看进度
    孩子学完后，家长想快速看看今天学了什么。
    → 需要：极简的进度卡片，一屏看完

    场景 4：家长晒成果
    孩子完成一个单元，家长想发朋友圈/家族群。
    → 需要：一键生成精美分享卡片

第三部分：产品理念与设计原则
====================================================================

3.1 品牌调性（四个关键词）

    纯粹 —— 打开就学字，学完就走，没有任何干扰
    极美 —— 每一屏都追求"家长想截图发朋友圈"的审美水准
    性价比 —— 让家长觉得"这个价格买到这个质量，赚了"
    护眼 —— 防沉迷和护眼不是附加功能，是品牌底线

3.2 视觉风格关键词

    - 星空主题：深蓝渐变底色、星星元素、微光粒子——呼应品牌名
    - 学习界面：暖白底色（#FFF8F0），护眼温和
    - 插画风格：高饱和度、治愈系、细腻质感（AI 生成后人工审核）
    - 字体：圆润儿童友好体，大号呈现，拼音辅助
    - 动效：弹性缓动为主，轻快不花哨

3.3 交互设计红线（所有页面必须遵守）

    □ 一屏只做一件事，信息密度极低
    □ 所有可点击元素最小触控区域 48×48pt
    □ 每个操作都有即时视听反馈（≤100ms 响应）
    □ 不出现任何文字菜单/导航——孩子不识字，全靠图形和颜色引导
    □ 错误反馈温和，绝不使用"错了""失败"等负面文案

4.1 页面总览

    ┌─────────────────────────────────────────────────────┐
    │                   小程序入口                         │
    │                      │                               │
    │              ┌───────┴───────┐                       │
    │              ▼               ▼                       │
    │         【首页】         【家长中心】                 │
    │         (学习地图)       (需验证进入)                 │
    │           │                  │                       │
    │     ┌─────┼─────┐      ┌────┼────┐                  │
    │     ▼     ▼     ▼      ▼    ▼    ▼                  │
    │  [单元入口] [今日复习]  [进度] [设置] [分享]         │
    │     │      (v1.1)                                    │
    │     ▼                                                │
    │  【学习流程】                                        │
    │     │                                                │
    │     ├── 步骤1：认（认字页）                          │
    │     ├── 步骤2：写（书写页）                          │
    │     ├── 步骤3：练（练习页）                          │
    │     ├── 小组练习（小组完成后）                       │
    │     └── 单元测验（全部小组完成后）                   │
    │            │                                         │
    │            ▼                                         │
    │      【测验结果页】→ 解锁下一单元 / 重考             │
    │                                                      │
    │      【付费引导页】（第二单元起触发）                 │
    │                                                      │
    │      【分享卡片页】（单元完成/里程碑触发）           │
    └─────────────────────────────────────────────────────┘

4.2 各页面职责简述

    P01 - 首页（学习地图）
      纵向卷轴星空地图，展示所有已上线单元。
      已完成→亮星，当前→跳动，未解锁→灰色锁。
      右上角"家长"入口按钮。
      （v1.1）"今日复习"入口 + 角标。

    P02 - 单元入口页
      展示该单元的主题名称、包含的字（以小卡片形式预览）、
      当前进度（第几组/共几组）。
      点击"开始学习"进入学习流程。

    P03 - 认字页
      上半屏：全幅字源插画
      下半屏：大号汉字 + 拼音 + 组词
      自动播放认读语音。点击可重播。
      底部"下一步"按钮（延迟出现）。

    P04 - 书写页
      田字格背景 + 灰色空心描红字。
      小手引导动画 → 孩子按笔顺书写。
      逐笔反馈（正确/错误）→ 全字完成特效。

    P05 - 练习页（单字速练，1 题）
      认+写完成后立即出 1 道题巩固该字。
      与 P06 共用题型组件，参数化控制题数。
      统一的顶部进度条 + 底部反馈区域。

    P06 - 小组练习页（2~3 题巩固）
      一组字学完后的 2~3 道巩固题。
      与 P05 共用题型组件，仅题数和触发时机不同。
      题目间无缝切换，全部答完显示本组结果（对/错数量）。

    P07 - 单元测验页
      全单元字学完后的综合测验。
      8~10 道题，涵盖多种题型，顶部带题目进度条。
      完成后进入测验结果页。

    P08 - 测验结果页
      星级展示（1~3 星）+ 大字鼓励语。
      通过：展示"下一单元已解锁"+ "去宝库"按钮。
      未通过：展示"再试一次"（可重考）。

    P09 - 付费引导页
      第二个单元起，未付费用户触发。
      展示产品价值 + 当前可用的付费选项。
      简洁明了，3 秒看懂。

    P10 - 小星宝库（成就收藏 + 分享）
      孩子已学过的字 / 已得到的星星 / 已解锁的成就，按主题归类展示。
      点击单个成就可生成精美分享卡片（保存相册 / 分享微信）。
      入口：P01 首页右下角金色角标 / P08 通关后"去宝库"按钮。
      （PRD v3.0 原"分享卡片独立页"已并入此页，2026-04-20 决策）

    P11 - 家长中心
      入口需通过数学验证题（如"3+5=?"）。
      包含三个子页面：学习进度、设置、分享记录。

    全局组件 · 过关庆祝
      P05/P06 答对后触发的全屏弹层（撒花+鼓励语+1.5秒后自动消失）。
      不占独立页面编号。视觉复用原 DWBnB 设计稿。

    全局组件 · 上下文引导
      首次进入 P03/P04/P05 等关键页时，在按钮旁出现小手指动画 +
      短句提示（"点一下听声音"/"跟着小手划一划"）。
      不占独立页面编号。
      （PRD v3.0 原"P12 操作引导独立页"已废弃，2026-04-20 决策：
       3-6 岁儿童不阅读说明书，引导必须在上下文中出现）

第五部分：核心学习流程详细设计
====================================================================

5.1 总体流程

    ┌──────────────────────────────────────────────┐
    │  进入单元                                     │
    │    │                                          │
    │    ▼                                          │
    │  ┌─────── 循环：每组 N 个字 ───────┐         │
    │  │  ┌── 循环：组内逐字 ──┐         │         │
    │  │  │  认 → 写 → 练      │         │         │
    │  │  └── 下一个字 ────────┘         │         │
    │  │         │                        │         │
    │  │         ▼                        │         │
    │  │    小组练习（2~3 题）             │         │
    │  │         │                        │         │
    │  │         ▼                        │         │
    │  └─── 下一组 ──────────────────────┘         │
    │              │                                │
    │              ▼                                │
    │        单元综合测验（8~10 题）                 │
    │              │                                │
    │         ┌────┴────┐                           │
    │         ▼         ▼                           │
    │      通过       未通过                        │
    │    (1~3星)    ("再试一次")                    │
    │       │                                       │
    │       ▼                                       │
    │  解锁下一单元 + 触发分享卡片                  │
    └──────────────────────────────────────────────┘

5.2 步骤 1："认"（认字页）

    目的：3~5 秒内建立"字形 ↔ 字音 ↔ 字义"的直觉关联。

    界面布局：
    ┌──────────────────────┐
    │                      │
    │   ┌──────────────┐   │
    │   │              │   │
    │   │  字源插画     │   │ ← 上半屏，全幅，圆角卡片
    │   │  (全幅高清)   │   │
    │   │              │   │
    │   └──────────────┘   │
    │                      │
    │       火              │ ← 大号汉字（≥72pt）
    │      huǒ             │ ← 拼音（带声调）
    │     火焰的火          │ ← 组词+造句语境
    │                      │
    │              ┌─────┐ │
    │              │下一步│ │ ← 延迟 3 秒出现，呼吸动画
    │              └─────┘ │
    └──────────────────────┘

    交互细节：
    - 页面载入时：自动播放认读语音"火，火焰的火"
    - 点击插画或汉字区域：插画微缩放弹性动画（scale 0.95→1.05→1），
      重播语音
    - "下一步"按钮延迟 3 秒出现（防止孩子无脑快速跳过），
      出现后带呼吸缩放动画引导注意力
    - 点击"下一步"：页面向左滑出，进入"写"

5.3 步骤 2："写"（书写页）

    目的：通过正确笔顺的手指运动强化字形肌肉记忆。

    界面布局：
    ┌──────────────────────┐
    │       火  huǒ        │ ← 顶部小字提示当前字
    │  ┌──────────────────┐│
    │  │  ┌──┬──┐        ││
    │  │  │  │  │ 田字格  ││ ← 中央主体区域
    │  │  ├──┼──┤        ││
    │  │  │  │  │        ││
    │  │  └──┴──┘        ││
    │  │   灰色空心描红字  ││
    │  │   + 小手引导动画  ││
    │  └──────────────────┘│
    │                      │
    │  进度：●●○○  (笔画)  │ ← 底部笔画进度指示
    └──────────────────────┘

    交互细节：
    - 初始状态：灰色空心字，小手 Icon 从第一笔起点滑到终点，
      循环播放引导动画，直到孩子开始触摸
    - 孩子手指触摸并滑动：
      · 判定正确：笔画填充流光渐变色 + 水滴音效"叮"
      · 判定错误：笔画闪烁红色（0.3s）+ 设备轻微震动 + 低沉提示音"嘟"
        → 该笔画重置，小手重新引导
    - 容错机制：同一笔连续错 3 次 → 自动播放慢速示范动画（1.5 倍
      时长），然后该笔画进入"辅助模式"（放宽判定阈值）
    - 全字完成：
      · 字体颜色变为金色
      · 全屏粒子撒花特效（1.5 秒）
      · 播放鼓励语音"太棒啦！"
      · 自动延迟 1 秒后进入"练"

5.4 步骤 3："练"（单字速练）

    目的：学完认和写后，立即用一道题巩固这个字。

    每个字学完"写"之后，出一道快速练习题（从题型 A/B/C 中随机）。
    答对 → 进入下一个字的"认"。
    答错 → 显示正确答案 + 重播该字的认读语音 → 进入下一个字。

    这一步节奏要快（单题 5~10 秒），不做重度测验，只是"打一针
    加强记忆"。

第六部分：题型池设计
====================================================================

题型池是一个持续扩充的"菜单"。每个单元从中选择适合的题型，
不要求全部使用。

6.1 题型 A：拖拽匹配（核心题型，首单元启用）

    ┌──────────────────────┐
    │  ┌──────────────┐    │
    │  │  插画         │    │ ← 目标图片
    │  │  (如：火焰)   │    │
    │  └──────────────┘    │
    │                      │
    │  ⓐ 水  ⓑ 火  ⓒ 木   │ ← 3 个汉字气泡，可拖拽
    └──────────────────────┘

    - 玩法：拖拽正确汉字到插画上
    - 正确：气泡爆炸消散 + 撒花 + "叮"音效
    - 错误：弹簧回弹动画 + 轻震 + "嘟"音效
    - 干扰项：从同组/同单元其他字中选取

6.2 题型 B：听音选字（首单元启用）

    ┌──────────────────────┐
    │                      │
    │     🔊 播放语音       │ ← 自动播放，可点击重播
    │    "shuǐ，水流的水"  │
    │                      │
    │  ┌───┐ ┌───┐ ┌───┐  │
    │  │ 水 │ │ 冰 │ │ 木 │  │ ← 3~4 个汉字卡片
    │  └───┘ └───┘ └───┘  │
    └──────────────────────┘

    - 玩法：听语音，点选正确汉字
    - 强化"音→形"关联
    - 可点击喇叭图标重播语音

6.3 题型 C：看图选字（首单元启用）

    ┌──────────────────────┐
    │  ┌──────────────┐    │
    │  │  插画         │    │
    │  │  (如：苹果)   │    │
    │  └──────────────┘    │
    │                      │
    │  ┌───┐ ┌───┐ ┌───┐  │
    │  │ 果 │ │ 木 │ │ 大 │  │ ← 点选（非拖拽）
    │  └───┘ └───┘ └───┘  │
    └──────────────────────┘

    - 与拖拽匹配类似，但交互为纯点击
    - 更快节奏，适合测验和复习场景

6.4 题型 D：字形拼合（后续单元引入，适合合体字）

    ┌──────────────────────┐
    │                      │
    │    拼出这个字：明     │
    │                      │
    │     [日]    [月]     │ ← 部件散落，拖拽拼合
    │          [木]        │ ← 干扰部件
    │                      │
    │      ┌────┐          │
    │      │ 目标区 │       │ ← 拖到此处拼合
    │      └────┘          │
    └──────────────────────┘

    - 将合体字拆成 2~3 个部件 + 1 个干扰部件
    - 帮助理解汉字结构（形声字、会意字）
    - 预计第 3~5 单元开始出现合体字时引入

6.5 题型 E：找不同（后续单元引入，适合易混淆字）

    ┌──────────────────────┐
    │                      │
    │    找出不一样的那个    │
    │                      │
    │  ┌───┐ ┌───┐        │
    │  │ 大 │ │ 大 │        │
    │  └───┘ └───┘        │
    │  ┌───┐ ┌───┐        │
    │  │ 犬 │ │ 大 │        │ ← 点击"犬"
    │  └───┘ └───┘        │
    └──────────────────────┘

    - 4 个汉字中混入 1 个形近干扰字
    - 强化字形辨析（如 人/入、土/士、大/犬）

6.6 后续可探索的题型方向（未定，视迭代需要引入）

    - 连线题（字↔图 / 字↔拼音）
    - 填空题（简单句子中选字）
    - 语境选择（场景图+选字）
    - 跟读题（语音识别孩子发音，技术可行性待验证）

第七部分：单元制与学习地图
====================================================================

7.1 单元定义

    - 单元是产品的最小交付和最小付费粒度。
    - 每个单元包含一组汉字，字数不固定（可能 10~20 字），
      根据汉字的语义关联、对比关系、难度搭配自然分组。
    - 每个单元是一次独立的"设计→开发→上线→反馈"闭环。
    - 单元之间允许玩法差异——后续单元可引入新题型和新交互。

7.2 单元内部结构

    单元
    ├── 小组 1（3~5 字）
    │     ├── 字 1：认 → 写 → 练
    │     ├── 字 2：认 → 写 → 练
    │     ├── ...
    │     └── 小组练习（2~3 题）
    ├── 小组 2（3~5 字）
    │     └── ...
    ├── ...（更多小组）
    └── 单元综合测验（8~10 题）
          └── 结果：1~3 星评价

    - 小组练习：从本组字中出题，巩固短期记忆
    - 单元综合测验：从全单元字中交叉出题，检验整体掌握
    - 通过标准：答对 60% 以上（如 8 题对 5 题）
    - 未通过可无限重考，每次随机重新组题

7.3 字库规划

    一期目标：1300 字（对标洪恩识字字库范围）
    远期目标：1800 字（对标部编版小学语文 + 现代汉语常用字表，待一期上线后评估是否继续扩字库）
    交付方式：渐进式，一个单元一个单元上线
    备注：1800 数字原为初期设定无严谨依据（2026-04-22 修订），实际是否扩到 1800 由一期数据回流决策
    选字优先级：
      早期 → 象形字、高频生活字、笔画少
      中期 → 形声字、会意字、常用扩展字
      后期 → 低频课标字、易混淆字辨析
    具体分配不提前锁死，每个单元设计时根据已有内容和反馈决定。

7.4 学习地图（首页）

    视觉：纵向卷轴，星空主题背景

9.2 定价细则

    层级 1：免费
    - Unit-01 完全免费，零门槛体验
    - 这个单元的质量直接决定后续转化率

    层级 2：低价破冰（9.9 元）
    - 进入 Unit-02 时弹出付费引导 + 新用户优惠券
    - 目标：建立付费关系，让家长"先掏一次钱"
    - 这一步的转化率是整个商业模型的关键指标

    层级 3：后续单元
    - 价格暂不锁定
    - 方向：按单元单独购买 / 按阶段打包，灵活组合
    - 根据付费数据和用户反馈迭代定价策略

    层级 4：终身买断（299 或 399 元）
    - 解锁全部已上线 + 未来更新的所有内容
    - 对比洪恩仍有显著性价比优势
    - 具体定价在首批用户数据出来后最终确定

9.3 付费引导页设计原则

    - 3 秒内让家长看懂"我需要付多少钱、能得到什么"
    - 突出免费单元中孩子已取得的成果（"宝宝已经学会了 X 个字！"）
    - 不使用压迫性倒计时或虚假折扣
    - 提供"稍后再说"选项（返回首页，不强制付费）


====================================================================
第十部分：离线与缓存策略
====================================================================

10.1 设计目标

    在网络不稳定时，保证孩子当前正在学的内容不中断。

10.2 预加载策略

    - 每次进入小程序时，自动预加载：当前单元 + 下一个单元的
      全部资源（插画、音频）
    - 用户在 WiFi 环境下，额外预加载下下个单元（如有）

10.3 缓存机制

    - 学习流程中所有资源优先读本地缓存
    - 缓存未命中时请求 CDN
    - 资源下载失败时使用占位图/默认音频，不阻断学习流程

10.4 离线降级

    - 完全断网 + 已缓存：正常学习，无感知
    - 完全断网 + 未缓存：友好提示"连上网络后就能学这个单元啦"，
      引导孩子回到已缓存的单元
    - 弱网：加载中展示趣味等待动画（不要空白屏）

10.5 存储控制

    - 单个单元资源包预估：5~10 MB（视字数和插画尺寸）
    - 同时最多缓存 3 个单元（约 15~30 MB）
    - 超出时自动清理最早的已完成单元缓存
    - 学习进度数据始终保留在本地（极小，几 KB）

第十二部分：AI 内容量产工作流
====================================================================

每个单元上线前执行以下流水线：

12.1 数据配置（Claude Opus 4.6）

    为该单元每个字生成 JSON 配置：
    {
      "char": "火",
      "pinyin": "huǒ",
      "word": "火焰",
      "sentence": "火焰的火",
      "stroke_count": 4,
      "unit_id": "unit-01",
      "group_id": "group-03",
      "illustration_prompt": "水墨风格的跳动火焰，暖色调，...",
      "quiz_config": {
        "distractors_from": ["水", "木", "土"],
        "applicable_types": ["drag_match", "listen_pick", "image_pick"]
      }
    }

12.2 插画资产（Nano banana pro）

    - 根据 illustration_prompt 生成插画
    - 统一尺寸：750×750px（适配小程序屏宽）
    - 人工审核每张图：风格一致性、内容准确性、儿童友好度
    - 不合格的重新生成

12.3 音频资产（AI 语音合成）

    - 认读语音：每字一条，格式"X，XX 的 X"
    - 音色要求：清晰、温暖的儿童友好音色
    - 统一音量标准化处理
    - 通用音效库（一次性制作，全局复用）：
      · 正确：水滴音"叮"
      · 错误：低沉提示"嘟"
      · 完成：撒花音效
      · 鼓励语："太棒啦！" / "你真厉害！" / "继续加油！"

12.4 质量把控

    每个单元的资产必须过以下检查：
    □ 插画风格与已上线单元一致
    □ 音频音量、音色一致
    □ JSON 配置字段完整，无错漏
    □ 干扰项选择合理（不会产生歧义）
    □ 在真机上实际跑一遍完整流程


====================================================================
第十三部分：Unit-01 完整设计
====================================================================

这是产品的第一个可交付单元，也是免费体验单元。
它的质量直接决定家长是否愿意为 Unit-02 付费。

13.1 基本信息

    单元编号：Unit-01
    单元名称：启蒙篇·第一单元（正式名称待定，可用主题命名如"天地万物"）
    定位：免费体验单元
    目标：让孩子觉得好玩，让家长觉得有效且精美

13.2 选字（初选，待最终确认）

    选字标准：
    - 最高频生活常用字
    - 优先象形字（插画视觉冲击力强）
    - 笔画 1~8 画
    - 字与字之间有对比性（便于出干扰项）

    初选 20 字：
    一、二、三、大、小、上、下、人、口、手
    日、月、水、火、山、木、土、天、目、耳

    说明：最终字数和具体选字根据实际分组效果可调整，不一定
    严格 20 个。

13.3 分组

    第 1 组（5 字）：一 二 三 大 小
      → 数字+大小对比，笔画极少，建立"我能写对"的信心

    第 2 组（5 字）：上 下 人 口 手
      → 方位+身体，生活高频，字形差异明显

    第 3 组（5 字）：日 月 水 火 山
      → 自然元素，最经典的象形字，插画最容易出彩

    第 4 组（5 字）：木 土 天 目 耳
      → 自然+身体收尾，与前三组形成呼应

13.4 启用题型

    从题型池选择 3 种：

    ✓ 题型 A - 拖拽匹配（主力题型，每组必出）
    ✓ 题型 B - 听音选字（每组出 1 道）
    ✓ 题型 C - 看图选字（综合测验中使用）

    ✗ 题型 D - 字形拼合（首单元全是独体字，不适用）
    ✗ 题型 E - 找不同（首单元字形差异大，太简单）

13.5 小组练习配置

    每组 5 字学完后：
    - 3 道题：2 道拖拽匹配 + 1 道听音选字
    - 干扰项从本组 5 字中选取

13.6 单元综合测验配置

    全部 4 组学完后：
    - 8 道题：3 道拖拽匹配 + 2 道听音选字 + 3 道看图选字
    - 干扰项从全单元 20 字中交叉选取
    - 通过标准：答对 5 道及以上（≥62.5%）
    - 评星：5~6 对→1 星，7 对→2 星，全对→3 星
    - 未通过 → 鼓励语"再来一次吧"，可重考（重新随机组题）

13.7 特殊设计（仅首单元）

    - 第一个字（"一"）进入前：触发操作引导动画
      · 认字页引导："点一下，听声音哦"
      · 书写页引导："跟着小手，划一划"
    - "一"只有 1 笔，写完特效可以做得比其他字更夸张（全屏金色
      粒子爆炸），给孩子最大的"第一次成功"冲击
    - 第 3 组（日月水火山）的插画质量拉满——这组字最容易出
      惊艳的视觉效果，是截图传播的核心素材
