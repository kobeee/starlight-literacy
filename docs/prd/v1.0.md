【第一部分：核心推导路径（我们是怎么走到这一步的？）】

在决定做这款产品之前，我们经历了一系列严密的商业与逻辑推演，最终锁定了现在的MVP形态。推导路径如下：

1. 起心动念（洞察痛点）：
   - 现象：“洪恩识字”体验好但价格昂贵（永久包两三百元），朋友及部分家长觉得太贵。
   - 直觉：做一个平替版，依靠低价抢占下沉或对价格敏感的市场。

2. 第一次排雷（摒弃 1:1 像素级复刻）：
   - 洪恩的壁垒在于庞大的3D美术资产和上百人的工业化产能。
   - 结论：独立开发者绝不能碰重度3D和庞大游戏分支，必须发挥“船小好掉头”和2026年最新AI工具的优势。

3. 第二次排雷（摒弃“实时大模型伴读”与“复杂探索”）：
   - 曾设想用大模型做实时生成互动和庞大的探索地图。
   - 否定原因：1. API算力成本极高，无法盈利；2. 实时生成面临极高的国内儿童内容合规风险；3. 3岁低幼儿童认知有限，看不懂复杂地图和长对话，只需要高频的视听刺激。

4. 商业模式顿悟（微型 SaaS 的降维打法）：
   - 目标校准：不需要颠覆行业，不需要大体量。只要伺候好 500 个掏钱的家长，项目就算成功。
   - 营销逻辑：立住“嫌洪恩贵，自己给娃写了个护眼纯净版”的硬核人设，依靠小红书和社群做极低价（29.9元）冲动消费转化。

5. 最终形态确立（AI工业化预制 + 微信小程序极简复刻）：
   - 不搞花哨的游戏，只复刻洪恩最核心的、家长最看重的“认、写、练”三步法闭环。
   - AI 的作用不是“前台噱头”，而是“后台廉价的外包美术和程序员”。批量生成静态资产，打包成轻量级小程序。


【第二部分：产品设计方案 (PRD)】

一、 产品定位与商业设计
1. 定位：基于微信小程序的“反网游化、纯净极简、高颜值”儿童识字平替工具。
2. 对外卖点：
   - 纯粹：没有花哨游戏和宠物养成，专注识字，护眼防沉迷。
   - 极美：采用顶级插画审美（AI降维赋能）。
   - 极致性价比：洪恩 1/10 的价格，买断制不套路。
3. 变现模式：
   - 免费诱饵：第 1-20 个字完全免费体验。
   - 付费墙：点击第 21 个字弹出付费，29.9 元永久买断首发 300 个核心高频字。

二、 核心交互体验（极简三步法）
剥离所有冗余游戏，每个字严格执行以下三个闭环交互：

模块 1：“认”（平替洪恩的3D字源动画）
- 界面：大字块 + 拼音。
- 视觉呈现：底层为一张 Nano banana pro 生成的极美“字源插画”（如“火”字配绝美的水墨火焰图）。
- 听觉反馈：进入即播放童声 TTS “火，火焰的火”。
- 交互：点击图片或字块，图片微缩放（CSS弹性动画），重播音效。停留3秒后自动显示“下一步”。

模块 2：“写”（平替洪恩的笔画描红）
- 界面：纯净的田字格背景，中心为灰色空心汉字。
- 底层技术：接入开源库 `Hanzi Writer`（配合 Canvas 2D）。
- 交互：小手 Icon 提示第一笔。孩子需按正确笔顺滑动。
- 反馈系统：
  - 正确：笔画瞬间填满流光颜色，播放“叮（水滴声）”。
  - 错误：笔画爆红闪烁，屏幕微震（调用小程序震动API），播放“嘟”声。
- 完成：全字写完，播放全屏特效（Lottie粒子）及“太棒啦”语音。

模块 3：“练”（平替洪恩的复杂小游戏）
- 统一题型：拖拽匹配（Drag & Drop）。
- 界面：上方展示图片（如水流插画），下方给出 3 个备选汉字泡泡（如“水”、“冰”、“木”）。
- 底层技术：使用小程序 WXS（WeiXin Script）实现 60fps 无延迟丝滑拖拽。
- 交互反馈：拖对爆炸消除+撒花特效；拖错弹簧回退。

三、 技术架构选型（2026适配版）
1. 前端框架：Uni-app (Vue 3) 或 Taro (React)，主包严控 < 2MB。
2. 动画引擎：
   - CSS3 Keyframes 处理 UI 弹跳与呼吸。
   - Lottie-miniprogram 处理通用的烟花、撒花特效。
3. 资源托管（极速省钱）：所有由 AI 生成的高清图、MP3 音效全部存放在云端 CDN，小程序动态拉取，实现秒开且不占包体。

四、 AI 工作流与内容量产 SOP（您的执行手册）
周末 48 小时量产 300 个字的闭环打法：
1. 数据层 (Opus 4.6)：输入 Prompt 批量生成 300 个字的 JSON 配置表（含汉字、拼音、组词、对应生图 Prompt）。
2. 资产层 (Nano banana pro)：根据 Opus 的 Prompt，花费几十美元极速跑出 300 张高质量配套插图。
3. 声音层：调用免费或廉价的云端 TTS API，生成 300 个认读 MP3。
4. 开发层 (Opus 4.6/Claude Code)：由大模型直接生成上述三大模块的微信小程序前后端代码，人工只做组件拼装和 WXS 拖拽调试。

五、 冷启动推广执行
1. 首批种子：直接把小程序码发给身边嫌洪恩贵的朋友，收集第一波真实设备运行反馈。
2. 平台截流：
   - 闲鱼：挂商品《绝美纯净版识字小程序，29.9永久买断》。
   - 小红书：发布笔记《洪恩200多太贵？程序员爸爸一气之下给女儿写了个平替，爆肝周末纯净无游戏》，引流私域/直接扫码。
